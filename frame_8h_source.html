24 // With the SIMPLE_FRAME flag, the size of frames cannot be changed at
25 // runtime. This is an issue for runnable-at-a-time mode, where global
26 // variables can be continually added. To handle this, the frame for
27 // global variables is an "indirect" frame. It holds one variable, which is
28 // a link to another frame. When the subframe is too small, a larger
29 // runtime array is allocated, and the link is changed.
